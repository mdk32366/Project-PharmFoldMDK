## 2. The finding that reframed the day — the rerun was blocked, and nobody knew

The pre-work sequenced the rerun as *"`core.enqueue` is idempotent; confirm it offers exactly 5."*
**Reading the code falsified that.** Enqueue idempotency keys on the **`protein_analyses` row**
(`core/enqueue.py:123`) — and all five failed targets already have one from the first attempt. So a
plain `enqueue --bucket rental` reports them **`existed`** and creates **zero** jobs.

Their `jobs` rows were stuck non-`pending` (the owner confirmed **all five `failed`**), the worker
claims only `pending`, and — the load-bearing catch — **`reap_stale` has no production caller**
(verified: no Fly cron, no app-startup hook, no claim-path reap; it runs only in tests). So there
was **no path back to `pending`, manual or automatic, short of hand-run SQL against prod.** A plain
rent-and-run would have booted a paid pod, polled `204 empty`, and folded nothing.

**That is why D-044 exists.** The re-fold is a real operational capability the system was missing, not
a convenience — and it is exactly the kind of by-hand-against-prod step (the token-mangling class of
failure) worth removing from a runbook before it costs money.

## 3. The failed state — built to the mechanism, not to these five

D-043's component is built **blind to which rows fail**: the supplier emits three states from a
`jobs` join, tested against fixtures, and the UI renders **zero failures correctly**. It does not
hard-code ADAM17/IGF2R/NOTCH2/PTPRZ1/SDK1. This is deliberate — the rerun is about to overwrite
those five, and a component specced against an expiring state is the trap PREWORK §8 named. The
rerun will populate real reasons for free; until then the surface renders whatever the DB honestly
holds (for the five right now, whatever the crash left — likely a reaped marker, not the OOM text).

**One honest boundary preserved:** `claimed`/`pending` are in-flight, **not** failures — only a
terminal `failed` counts (D-030's status machine). A target being re-folded never flashes `failed`.
